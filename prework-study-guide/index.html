HTML

  • The head element contains information about the webpage.
  • The head element can contain information such as:
    • the title element, for the webpage title
    • the style element, for defining CSS styles
    • the base element, for defining the base URL
    • the link element, for connecting external resources
    • the script element, for embedding client-side scripts
    • and the meta element. It can store all sorts of metadata for the webpage, such as character sets, viewport sizes and authorship information.
  • The body element represents the visible content shown to the user.
  • The body is split into a header, main and footer.
  • Examples of body elements include:
    • p: text paragraphs
    • ul, ol, li: unordered lists, ordered lists and list items
    • h1-6: headings from biggest to smallest
    • img: images, with width, height, source and alt-text attributes
    • a: hyperlinks
    • and page separations like br and div.

CSS

  • The best practice is to keep all CSS rules in an external CSS file to keep concerns separated. This file must be linked to in the HTML head.
  • It is also possible to style individual elements inline, or embed rules into the HTML head.
  • CSS rules consist of the HTML element/s to style, followed by a JSON-styled dictionary of property: value.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Global rules can be defined using the wild card symbol (*).
  • Classes can be defined with syntax .new-class, and elements in the HTML file assigned to said class will be styled with the associated rules.

Git

  • git status: checks what branch we are currently on
  • git pull origin branch-name: updates our branch to include recent changes from other developers
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stages all changed files to be committed
  • git commit -m "message": commits all staged files with the message
  • git push origin branch-name: pushes all commits to origin
  • git merge branch-name: merges the branch into our current branch

JavaScript

  • JavaScript scripts are linked to in the page body, typically after the footer.
  • A variable is a named container that allows us to store data in our code.
  • JavaScript variables are dynamically and weakly typed: variable types don't have to be defined and they allow implicit type conversion.
  • Basic data types include Boolean, Number (includes floating point), String and Object, as well as utility types for Null and Undefined.
  • JavaScript debugging can be done by utilising console logging. The page console can be viewed with the browser devtools (Ctrl-Shift-I on Windows and Chrome).
  • Control flow is the order in which a computer executes code in a script.
  • Conditional structures (including if-else statements) alter the control flow by executing code if the provided condition returns a truthy value.
  • Use the strict comparison operator ("===") to check that the datatype matches as well as the value.
  • Arrays are sequential collections of related data items. Multitype arrays are permitted.
  • Arrays can be iterated over in iterative structures (including for loops).
  • Functions are blocks of code that are executed by calling them by name, allowing for separation of concerns and for repeated calls.
  • Functions can have parameters passed to them when called.
